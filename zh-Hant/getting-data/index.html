請求 Overture 資料

Overture 的六個資料主題 — 地址, base, 建築物, 區域, 地點, 以及 交通 — 可以在 Amazon S3 和 Microsoft Azure Blob Storage 這兩個位置免費獲取：

提供者位置
Amazon S3s3://overturemaps-us-west-2/release/
Azure Blob Storagehttps://overturemapswestus2.blob.core.windows.net/release/

最新的發布路徑為：

2024-07-22.0/

GeoParquet​

Overture 以 GeoParquet 格式分發其資料集，這是一種 column-oriented 的空間資料格式，是 Apache Parquet 的向後兼容擴充套件。Parquet (和 GeoParquet) 對 "cloud-native" 查詢進行了改良，這代表可以使用許多開發者友好的工具來有效地獲取雲端托管資料的 column "chunks"。如果對 GeoParquet 不熟悉，可以參考 這篇指南。

僅請求需要的資料（建議選項）​

強烈建議 使用 AWS 和 Azure 上可用的服務來獲得雲端資料，或者使用像 DuckDB 這樣的本地安裝查詢引擎。這樣可以讓您只下載所需的資料集。也可以使用 Overture 的實驗性 Python command-line tool 來按功能類型和感興趣區域下載資料。

  • Amazon Athena（需要有帳號）
  • Microsoft Synapse（需要有帳號）
  • DuckDB
  • Python command-line tool

如果我想要所有的資料怎麼辦？​

warning

確定嗎？所有檔案的總大小為 ~440 GB。

可以直接從 Azure Blob Storage 或 Amazon S3 下載 GeoParquet。以下是操作步驟：

在安裝 AWS CLI 之後，您可以使用以下指令從 S3 下載檔案。將 <DESTINATION> 設定為本地目錄路徑來下載檔案，或設定為您控制的 s3:// 路徑來將檔案複製到您的 S3 儲存貯體中。

aws s3 cp --region us-west-2 --no-sign-request --recursive s3://overturemaps-us-west-2/release/2024-07-22.0/ <DESTINATION>

使用 Azure Storage Explorer 或 AzCopy 指令從 Azure Blob Storage 下載檔案。下面是一個 azcopy 指令範例。

azcopy copy "https://overturemapswestus2.dfs.core.windows.net/release/2024-07-22.0/" "<<local directory path>>"  --recursive
